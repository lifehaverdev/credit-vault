## 🚨 Root Cause Analysis

### **The Issue**
The JavaScript implementation and the Solidity LibClone implementation are generating **different init code hashes**, indicating a fundamental difference in how the init code is constructed.

### **Key Differences Identified**

1. **Memory Layout Structure**: The LibClone implementation uses a specific memory layout that differs from what the JavaScript implementation expects.

2. **Byte Arrangement**: The exact byte arrangement in the init code is:
   - **Length prefix**: `0x610096` (150 bytes total length)
   - **Beacon address**: Embedded at specific offsets within the runtime code
   - **Runtime code**: Three 32-byte chunks of bytecode
   - **Args**: Encoded at the end

3. **Hash Calculation**: LibClone calculates the hash from a specific memory region (offset 0x16, length n + 0x75) rather than the entire init code.
